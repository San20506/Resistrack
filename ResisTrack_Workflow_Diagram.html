End-to-End Execution Flow
1
Patient Admission Event
Patient is admitted to hospital. EHR (Epic/Cerner) generates an ADT A01 (Admit) HL7 v2 message. LIS systems begin streaming lab order events. Pharmacy system records active medication history.
EHR LIS RIS HL7 ADT A01
2
Data Gateway & HL7 ‚Üí FHIR Transformation
Messages are received by the Data Gateway Aggregation Layer. Mirth Connect transforms HL7 v2 ADT/ORU/RDE messages into FHIR R4 resource bundles (Patient, Encounter, Observation, MedicationRequest, Condition).
Mirth Connect FHIR R4 HL7 v2 ‚Üí R4
3
Encrypted Transmission to AWS VPC
FHIR bundles are transmitted via TLS 1.3-encrypted channel with Mutual Certificate Authentication over a VPN / AWS Direct Connect private tunnel. The hospital DMZ connects exclusively to the private AWS VPC ‚Äî no public internet path.
TLS 1.3 Mutual Auth VPN / Direct Connect VPC Private
4
API Gateway Ingestion & Validation
AWS API Gateway + Lambda receives FHIR bundles. Lambda functions validate FHIR schema conformance, de-duplicate resources, and perform PHI tokenization. Valid bundles are routed to AWS HealthLake for persistence.
API Gateway Lambda Schema Validation De-duplication
5
Storage: HealthLake + RDS + S3
FHIR R4 patient resources stored in AWS HealthLake (AES-256 encrypted, HIPAA-eligible). Unstructured clinical notes stored in S3. Model outputs and audit logs written to Amazon RDS (PostgreSQL). All PHI encrypted at rest with AWS KMS.
HealthLake RDS PostgreSQL S3 AES-256 / KMS
6
Feature Engineering Pipeline
SageMaker Processing Jobs extract: 47 structured tabular features (lab values, vitals, prior antibiotics, comorbidities), 72-hour temporal time-series tensors of labs + vitals, and ClinicalBERT embeddings of the 3 most recent clinical notes.
SageMaker Processing Feature Store NLP Extraction Time-Series
7
AI/ML Ensemble Inference
Three models run in parallel on SageMaker: (a) XGBoost produces AMR Risk Score from tabular features, (b) PyTorch LSTM processes time-series vital/lab trends, (c) ClinicalBERT processes unstructured note embeddings. Outputs are ensemble-weighted and calibrated via Platt scaling to a final 0‚Äì100 risk score.
XGBoost PyTorch LSTM ClinicalBERT Ensemble SHAP
8
Access Control & Authorization Check
Before results are surfaced, OAuth 2.0 SMART on FHIR authorization validates the requesting EHR app and clinician identity. AWS IAM RBAC checks confirm the requesting user role has access to the patient's data scope. Access tokens validated via JWT verification.
OAuth 2.0 SMART on FHIR IAM RBAC JWT
9
CDS Hook Alert ‚Üí EHR Workflow
CDS Hooks service fires on patient-view and order-sign events. A CDS Card is injected into the clinician's EHR screen (Epic/Cerner) showing: AMR Risk Tier, top-3 SHAP-explained risk factors in plain language, and stewardship recommendation. Clinician can acknowledge, override, or escalate ‚Äî all actions logged.
CDS Hooks Epic / Cerner Inline Alert No New Login
10
Multi-Role Notifications
Parallel notifications dispatched based on risk tier: Pharmacy team alerted for High/Critical-risk antibiotic stewardship review. Infection Control Officer alerted for Critical-tier MDR organism flags. Hospital-wide trend analytics updated in real-time dashboard. All notifications contain de-identified patient tokens (no PHI in alerts).
Pharmacy Alert Infection Control De-identified SNS
11
Clinical Dashboard & Reporting
Authorized users access the ResisTrack web dashboard (React.js / CloudFront) via SMART launch or direct login. Dashboards show: ward-level AMR risk heatmaps, individual patient risk timelines, infection trend analytics, and weekly stewardship reports (PDF/CSV export). Role-based views limit each user to their authorized data scope.
React Dashboard CloudFront RBAC Views PDF Reports
12
Continuous Learning & Model Retraining
Clinician override actions and culture result confirmations are captured as model feedback. SageMaker Pipelines runs automated monthly retraining on new hospital data. Models pass automated regression tests against a held-out validation set before promotion to production via the SageMaker Model Registry and CodePipeline CI/CD.
SageMaker Pipelines Auto Retraining A/B Validation Model Registry
